# Contributing to TraceVault

First off, thank you for considering contributing to TraceVault! It's people like you who make this community-driven platform better for everyone.

## How Can I Contribute?

### Reporting Bugs
* Check the Issues tab to see if the bug has already been reported.
* If not, open a new issue with a clear title and a detailed description of the problem.

### Suggesting Enhancements
* Open an issue with the "enhancement" tag.
* Explain why this feature would be useful for the TraceVault community.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Ensure your code follows the existing style (we use ESLint and Prettier).
3. If you've added code that should be tested, add tests.
4. Ensure the test suite passes.
5. Open that PR!

## Style Guide
* **TypeScript:** Use strict types (no `any` where possible).
* **Components:** We use Tailwind CSS and Shadcn/UI for styling.
* **Commits:** Use conventional commits (e.g., `feat: add claim validation`).
